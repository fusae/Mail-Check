# 舆情报告新增功能说明

## 概述

本次更新为舆情报告系统增加了三个重要功能模块，使报告更加全面和实用。

## 新增功能

### 1. 情感分析与舆情态势

**位置：** 报告第七章

**功能说明：**
- **情感倾向分布**：将舆情分为正面/中性/负面三类，并统计各类占比
- **公众情绪指数**：综合评估公众情绪强度（0-100分）
- **情绪强度分析**：分析愤怒、悲伤、失望、质疑、担忧等具体情绪的强度
- **舆情热度趋势**：按日期展示舆情数量、平均风险分和热度指数的变化趋势

**数据示例：**
```
情感倾向分布：
- 负面：4条 (66.7%)
- 中性：2条 (33.3%)
- 正面：0条 (0%)

公众情绪指数：66.7/100 - 🟠 负面

情绪强度分析：
- 愤怒：75/100 (高)
- 失望：60/100 (高)
- 质疑：45/100 (中)
```

### 2. 舆情分类统计

**位置：** 报告第八章

**功能说明：**
- **问题类型分类**：自动将舆情分为8大类
  - 医疗质量
  - 服务态度
  - 收费问题
  - 环境卫生
  - 流程问题
  - 无障碍设施
  - 隐私保护
  - 其他
- **分类统计**：统计每类问题的数量、占比、平均风险分和严重程度
- **重点关注类型**：列出Top 3问题类型及典型案例
- **图表数据**：提供可视化图表所需的JSON数据

**数据示例：**
```
问题类型分布：
| 问题类型     | 数量 | 占比  | 平均风险分 | 严重程度 |
|-------------|------|-------|-----------|---------|
| 服务态度     | 2条  | 33.3% | 80.0      | 极高    |
| 收费问题     | 2条  | 33.3% | 45.0      | 中      |
| 无障碍设施   | 2条  | 33.3% | 60.0      | 高      |
```

### 3. 风险传播预测

**位置：** 报告第九章

**功能说明：**
- **传播趋势分析**：评估当前舆情的传播趋势（上升/平稳/下降）
- **未来3天预测**：预测未来3天的新增舆情数量、累计总数和平均风险分
- **未来7天预测**：预测未来7天的舆情发展态势
- **二次传播风险点**：识别可能引发二次传播的风险因素
  - 高风险事件发酵
  - 跨平台传播
  - 集中爆发
  - 情感极化
- **敏感时间节点**：提醒周末、节假日、月初月末等敏感时期
- **传播路径预测**：预测舆情可能扩散到的平台

**数据示例：**
```
未来3天预测：
- 新增舆情数量：9条
- 累计舆情总数：15条
- 预测平均风险分：66.0
- 置信度：中

二次传播风险点：
- 高风险事件发酵：存在1个高风险事件，可能引发媒体关注
- 跨平台传播：舆情已在2个平台传播，存在跨平台发酵风险

敏感时间节点：
- 2026-03-08 (周末)
  原因：周末期间用户活跃度高，舆情容易发酵传播
  建议：加强周末舆情监控
```

## 技术实现

### 核心方法

1. `_generate_sentiment_analysis_new(df)` - 生成情感分析数据
2. `_generate_category_statistics(df)` - 生成分类统计数据
3. `_generate_spread_forecast(df)` - 生成传播预测数据

### 报告格式化方法

1. `_format_sentiment_analysis_section(data)` - 格式化情感分析章节
2. `_format_category_statistics_section(data)` - 格式化分类统计章节
3. `_format_spread_forecast_section(data)` - 格式化传播预测章节

## 使用方法

### 生成报告

```python
from report_generator_enhanced import EnhancedReportGenerator
import pandas as pd

# 创建报告生成器
generator = EnhancedReportGenerator()

# 准备数据（DataFrame格式）
df = pd.read_csv('sentiment_data.csv')

# 生成报告数据
report_data = generator.generate_report_data(
    df=df,
    hospital_name='东莞市中医院',
    report_type='special'
)

# 生成Markdown报告
markdown_report = generator.generate_markdown_report(report_data)

# 保存报告
with open('report.md', 'w', encoding='utf-8') as f:
    f.write(markdown_report)

# 生成Word报告
generator.generate_word_report(report_data, 'report.docx')
```

### 访问新增数据

```python
# 情感分析数据
sentiment_analysis = report_data['sentiment_analysis_new']
print(f"公众情绪指数: {sentiment_analysis['public_emotion_index']}")
print(f"情感倾向: {sentiment_analysis['sentiment_ratio']}")

# 分类统计数据
category_stats = report_data['category_statistics']
print(f"问题类型总数: {category_stats['total_categories']}")
print(f"Top问题: {category_stats['top_categories']}")

# 传播预测数据
spread_forecast = report_data['spread_forecast']
print(f"3天预测: {spread_forecast['forecast_3days']}")
print(f"二次传播风险: {spread_forecast['secondary_risks']}")
```

## 数据要求

报告生成需要DataFrame包含以下字段：

**必需字段：**
- `标题` - 舆情标题
- `内容` - 舆情内容
- `来源` - 来源平台
- `创建时间` - 发布时间
- `风险分` - 风险评分（0-100）
- `严重程度` - 严重程度（high/medium/low）
- `状态` - 状态（active/inactive）

**可选字段：**
- `警示理由` - AI判断理由
- `科室` - 涉及科室
- `医院` - 医院名称
- `原文链接` - 原文URL

self.emotion_keywords = {

## 注意事项

1. **数据质量**：预测准确性依赖于历史数据的质量和数量
2. **置信度**：当数据量少于5条时，预测置信度会降低
3. **时间范围**：建议使用至少7天的数据以获得更准确的趋势分析
4. **中文分词**：如果安装了jieba库，情感分析会更准确
